# 边缘计算：端侧小模型与云端大模型的算力调度

## 核心问题
端侧模型（如 Gemini Nano、Llama Edge）如何在本地与云端大模型之间动态协作？"推理卸载"面临哪些工程挑战？

## 背景框架

### 1. 端侧模型的现状
- **Gemini Nano**：Google 已在 Pixel 设备部署，2-4B 参数，主打低延迟本地推理
- **Llama Edge**：Meta 的边缘开源方案，支持 WebAssembly 跑在浏览器/IoT设备
- **Apple Intelligence**：端侧小模型 + 云端 Private Compute Cloud

### 2. 协作模式
- **纯本地**：延迟最低，但能力受限于参数规模
- **纯云端**：能力无限，但受限于网络与成本
- **混合编排**（主流方向）：
  - LoRA Routing：小模型判断是否需要升级到云端
  - Speculative Decoding：小模型起草 + 大模型审核
  - Cascaded Inference：简单 query 本地答，复杂 query 上云

### 3. 工程挑战
- **决策何时卸载**：需要精准的"难度评估模型"
- **状态迁移**：上下文如何在端-云之间无缝传递
- **隐私悖论**：上传 partial context 是否真的比本地处理更安全？

## 核心观点

> **端侧小模型不是大模型的"精简版"，而是全新的推理范式。**
> 
> 当模型足够小（<3B），它可以在毫秒级响应，此时"是否需要调用大模型"本身就是一个小模型就能判断的事。这催生了 **"Model-as-a-Judge"** 的新架构。
